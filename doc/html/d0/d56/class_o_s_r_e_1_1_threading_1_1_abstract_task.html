Detailed Description

This abstract interface-class declares a task. Tasks can be used to handle several component jobs like streaming resource loader requests or the rendering. A task can work as a sequence. Then he will be started in the main thread context. Or the task can be started in a separate thread. You can install an own thread instance as well. To implement the behavior of the task you can install event-handlers. Each event with its data will be managed by a task job. TaskJobs will be queued and handled one after another. You can specify your own setup-method, if you want to add your own setup code.
